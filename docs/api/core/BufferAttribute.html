[name]

这个类存储了[page:BufferGeometry]相关数据。查看对应页面查看详细用法和示例。这个类通常用来储存内置属性顶点position，normals，color等等，不过存储[page:BufferGeometry]内自定义属性也是没有问题的。

构造函数

[name]( [page:TypedArray array], [page:Integer itemSize], [page:Boolean normalized] )

从相关的缓存区中实例化该属性。 itemSize为与指定顶点相关的数组值数量。 normalized表示缓存区内基础数据如何映射到GLSL着色器代码上。

属性

[property:TypedArray array]

存储与该属性相关的数据。这个元素应该有itemSize * numVertices个元素。这里numVertices是几何图形[page:BufferGeometry geometry]对应的顶点数。[page:TypedArray array]可以是UInt8Array, Int8Array, UInt16Array, Int16Array, 或 Float32Array的实例。

[property:Integer itemSize]

记录与顶点相关的数组数量。举个例子，如果一个属性有个三维向量（比如position, normal, 或 color），那么itemSize就是3。

[property:Integer count]

数组中元素的总数。

[property:Boolean needsUpdate]

标识该属性已被改变，需要重新发回GPU。当你修改数组的值后需要将其设为true。

[property:Boolean normalized]

表示缓存区内基础数据如何映射到GLSL着色器代码上。举个例子，如果[page:TypedArray array]是UInt16Array的实例，并且[page:Boolean normalized]为true，那么值0 - +65536将会对应GLSL属性上的0.0f - +1.0f。一个（有符号）Int16Array将对应-32767 - +32767到-1.0f - +1.0f。如果[page:Boolean normalized]为false，那么值将转换为浮点数，比如32767变为32767.0f。

[property:Integer version]

版本号，每当needsUpdate设为true时就会增加。

方法

[method:null copyAt] ( [page:Integer index1], attribute, [page:Integer index2] )

复制index2顶点数组中的itemSize值到index1中。

[method:null set] ( [page:Array value] )

从传递的数组中取值来设置相关数组。

[method:null setX]( index, x )

把数组中index * itemSize的值设为x。

[method:null setY]( index, y )

把数组中index * itemSize + 1的值设为y。

[method:null setZ]( index, z )

把数组中index * itemSize + 2的值设为z。

[method:null setXY]( index, x, y )

把数组中index * itemSize的值设为x， 把数组中index * itemSize + 1的值设为y。

[method:null setXYZ]( index, x, y, z )

把数组中index * itemSize的值设为x， 把数组中index * itemSize + 1的值设为y， 把数组中index * itemSize + 2的值设为z。

[method:null setXYZW]( index, x, y, z, w )

把数组中index * itemSize的值设为x， 把数组中index * itemSize + 1的值设为y， 把数组中index * itemSize + 2的值设为z， 把数组中index * itemSize + 3的值设为w。
